## 全体構成

### 必須セクション

各ノートブックは以下のセクションで構成されること：

```markdown
1. タイトルと概要
   - 明確な章番号とタイトル
   - この章で学ぶことの明示

2. 学習メタ情報
   - 推定学習時間（分単位）
   - 難易度（★☆☆☆☆ の5段階）
   - 前提知識（必要な前のノートブック）
   - カテゴリ（基礎/実践/応用など）

3. メインコンテンツ
   - セクションごとに明確な見出し
   - 理論 → 実装 → 検証 の流れ

4. まとめ
   - 学んだことの要約
   - 重要なポイントの再確認
   - 次のステップへの導線

5. サポート要素
   - よくあるエラーと解決法
   - 自己評価クイズ
   - 練習問題（推奨）
```

### ファイル命名規則

```
{番号}_{トピック}_{改善バージョン}.ipynb

例:
- 00_quick_start_improved_v2.ipynb
- 07_mlp_fundamentals_improved_v2.ipynb
- 15_titanic_eda_feature_engineering_improved_v2.ipynb
```

## コードの品質

### コードセルの基本ルール

1. **1セル1タスク原則**
   - 1つのコードセルで1つの明確なタスクを実行
   - 複数の処理を含む場合は分割を検討

2. **コメント密度**
   - 最低200行のコードに対して50行以上のコメント
   - すべてのコードブロックに説明コメント

3. **段階的実行**
   - 学習者が1セルずつ実行して理解できる構成
   - セル間の依存関係を最小化

### コメントの書き方

```python
# ============================================================
# [コードの説明]
# このセクションでは{目的}を行います
# ============================================================

# データの読み込み
# seabornに含まれるタイタニックデータセットを使用

# - データ型

# 統計サマリー
# 数値列の基本統計量（平均、中央値、標準偏差など）を確認

```python

### エラーハンドリング

```python
# 安全なデータ処理の例
# 欠損値があってもエラーにならないよう、事前にチェック
if df['age'].isnull().sum() > 0:
    print(f"欠損値を検出: {df['age'].isnull().sum()}件")
    # グループごとの中央値で補完
    for sex in ['male', 'female']:
        for pclass in [1, 2, 3]:
            mask = (df['sex'] == sex) & (df['pclass'] == pclass) & (df['age'].isnull())
            group_median = df[(df['sex'] == sex) & (df['pclass'] == pclass)]['age'].median()

            # グループに値がない場合のフォールバック
            if pd.isna(group_median):
                group_median = df['age'].median()

            df.loc[mask, 'age'] = group_median
```

- **入門レベル**: 5,000-8,000文字

{良い例}
```

**説明:**

---
```

**例：**
```markdown
### ⚠️ よくあるエラー #1: Shape Mismatch

ValueError: operands could not be broadcast together with shapes (100,) (50,)

**原因:**
1. 訓練データとテストデータのスケーリングを別々に実行
2. 特徴量の数が一致していない

**✅ 解決法:**

```python
# ❌ 問題のあるコード
scaler_train = StandardScaler()
X_train_scaled = scaler_train.fit_transform(X_train)

scaler_test = StandardScaler()  # 別のスケーラー！
X_test_scaled = scaler_test.fit_transform(X_test)

# ✅ 正しいコード
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)  # 訓練で学習
X_test_scaled = scaler.transform(X_test)        # テストに適用
```

**説明:**
訓練データで学習した変換を、テストデータにも適用する必要があります。
```

        'Hiragino Sans', 'Hiragino Maru Gothic Pro', 'AppleGothic',  # macOS

        'Noto Sans CJK JP', 'IPAexGothic', 'TakaoPGothic',  # Linux

## 各難易度レベルの特徴

### ★☆☆☆☆ 入門

- **対象**: プログラミング初心者
- **学習時間**: 30-60分
- **解説**: 非常に詳細、前提知識なし
- **コード**: 短く、1ステップずつ
- **数式**: 最小限または不要
- **例**: クイックスタート、基本的なデータ操作

### ★★☆☆☆ 初級

- **対象**: Pythonの基礎を理解している人
- **学習時間**: 60-120分
- **解説**: 詳細、基本概念は既知
- **コード**: 中程度、複数のステップ
- **数式**: 簡単な数式を含む
- **例**: データ前処理、基本的な可視化

### ★★★☆☆ 中級

- **対象**: 機械学習の基礎を理解している人
- **学習時間**: 90-150分
- **解説**: 詳細、理論的背景を含む
- **コード**: 複雑、複数のアプローチ
- **数式**: 数学的説明を含む
- **例**: モデル比較、ハイパーパラメータ調整

### ★★★★☆ 上級

- **対象**: 機械学習の実践経験がある人
- **学習時間**: 120-180分
- **解説**: 詳細、最新の手法を含む
- **コード**: 高度、最適化を含む
- **数式**: 詳細な数学的説明
- **例**: アンサンブル学習、モデル解釈

### ★★★★★ 実践

- **対象**: コンペティション参加者レベル
- **学習時間**: 180-400分
- **解説**: 包括的、実戦テクニック
- **コード**: プロダクションレベル
- **数式**: 必要に応じて
- **例**: Kaggleコンペ、エンドツーエンドプロジェクト
