# Jupyter Notebook 教育改善プロンプト

このプロンプトを使用して、学習用Jupyter Notebookを日本語の教科書フレームワークに基づいて改善できます。

---

## 📋 改善依頼プロンプト

```
以下の日本語教科書フレームワークに基づいて、このリポジトリ内の全てのJupyter Notebookを改善してください。

## 改善要件

### 1. 必須要素の追加

各ノートブックに以下を追加：

**メタ情報**:
- 学習目標（チェックボックス形式で3-5項目）
- 前提知識（依存関係を明記）
- 推定学習時間（分単位）
- 難易度（★1-5で表示）
- カテゴリー分類

**教育コンテンツ**:
- 詳細な日本語の説明（各セクション前に追加）
- 実世界での応用例（具体的な業界・用途）
- わかりやすいアナロジー（料理、スポーツなど日常的な例え）
- コード内の詳細なコメント（200行以上、各行の意味と理由を説明）

**学習支援**:
- よくあるエラーとその解決法（2-3個）
- ベストプラクティス
- 自己評価クイズ（2-5問、詳細な解説付き）
- 次のステップ（関連ノートブックへのリンク）

### 2. コンテンツ構造（Golden Pattern）

各ノートブックを以下の構造に：

1. **Why（なぜ）**: モチベーションと重要性
2. **What（何を）**: 概念の詳細説明
3. **How（どのように）**: 実装方法とコード
4. **Practice（実践）**: 演習とクイズ
5. **Summary（まとめ）**: 要点と次のステップ

### 3. 実装手順

**Phase 1: 分析**
1. 全ノートブックの現状を分析
2. 改善計画を策定
3. パイロット実装（1つのノートブックで試す）

**Phase 2: 実装**
1. パイロットのフィードバックを反映
2. 全ノートブックに適用
3. 詳細な日本語説明を追加

**Phase 3: 品質保証**
1. プレースホルダーの検出と削除
2. 一貫性の確認
3. リポジトリの整理

### 4. 重要な制約

**禁止事項**:
- ❌ 進捗管理機能（チェックボックスUI、進捗バーなど）は不要
- ❌ プレースホルダー（[ここに説明を記述]など）を残さない
- ❌ 元のコードの動作を変更しない

**必須事項**:
- ✅ コード内に200行以上の詳細なコメントを追加
- ✅ 各概念の前に詳しい日本語の説明を追加
- ✅ 実例と数値例を豊富に含める
- ✅ ユーザーが概念学習に集中できるよう、コード実装の詳細はコメントで説明

### 5. リポジトリ整理

改善完了後、以下の構造に整理：

```
repository/
├── notebooks/              # 改善版ノートブック（v2版のみ）
├── scripts/
│   ├── notebook_improvements/  # 改善用スクリプト
│   └── examples/              # サンプルコード
├── README.md              # 包括的なプロジェクト説明
└── ML_LEARNING_PLAN.md   # 学習計画（オプション）
```

**README.mdに含める内容**:
- プロジェクト概要
- ディレクトリ構造
- 学習カリキュラム（推定時間・難易度付き）
- インストール手順
- 使い方のヒント
- 技術スタック

### 6. 期待される成果

改善後、各ノートブックは以下を満たす：

**量的な目標**:
- セル数: +30-50%増加
- 日本語説明: 各ノートブック10,000文字以上
- コードコメント: 200行以上

**質的な目標**:
- 初学者が独学で理解できる
- 実世界の応用がイメージできる
- よくあるエラーを自己解決できる
- 次に学ぶべきことが明確

## 実行方法

1. まず、全ノートブックを確認して現状分析と改善方針を提案してください
2. 承認後、パイロット実装を1つのノートブックで実施してください
3. フィードバックを反映後、全ノートブックに適用してください
4. 最後にリポジトリを整理してください

段階的に進め、各フェーズで確認を取りながら作業してください。
```

## 🎯 使用方法

### ステップ1: プロンプトのカスタマイズ（必要に応じて）

リポジトリの特性に応じて、以下を調整：
- ノートブックのトピック（機械学習、データサイエンス、Pythonなど）
- 対象レベル（初心者、中級者、上級者）
- 特殊な要件（特定のライブラリ、フレームワークなど）

### ステップ2: Claude Codeでの実行

```bash
# リポジトリに移動
cd your-notebook-repository

# Claude Codeを起動し、上記のプロンプトを貼り付け
```

### ステップ3: 段階的な実行

1. **分析フェーズ**: Claudeがノートブックを分析し、計画を提案
2. **パイロットフェーズ**: 1つのノートブックで試験実装
3. **フィードバック**: 結果を確認し、必要に応じて調整を指示
4. **本実装**: 全ノートブックへの適用を承認
5. **整理**: リポジトリの整理を承認

## 🔧 トラブルシューティング

### よくある問題

**問題**: 改善後のノートブックが大きすぎる
**解決**: セル数の増加目標を調整（例: +20-30%に制限）

**問題**: 既存のコードが複雑でコメントが難しい
**解決**: コメント量の目標を調整、または段階的に追加

**問題**: トピックが専門的すぎてアナロジーが難しい
**解決**: 技術的な説明に重点を置き、可能な範囲でアナロジーを追加
